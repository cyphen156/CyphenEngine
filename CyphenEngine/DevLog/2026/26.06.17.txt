작업 요약
	Renderer 진입 전 File / FileSystem 단일 작업 I/O 범위를 마감했습니다.

	File에는 append 계열을 추가하고,
	FileSystem에는 파일 생성, 크기 조회, 복사, 이동,
	디렉터리 트리 생성 / 재귀 삭제 API를 추가했습니다.

	또한 Path 내부에 고정되어 있던 구분자 처리 기준을
	Separator / Separators 정책 계층으로 분리했습니다.

	CoreIoTests를 확장하여 최종적으로 PASS=69 / FAIL=0을 확인했습니다.


진행한 커밋
	#1_17 [Core] File append 및 FileSystem 단일 작업 API 마감


주요 정리 내용
	1. File append API 추가
		File에 AppendAllBytes / AppendAllText를 추가했습니다.

		AppendAllBytes는 파일 끝에 raw byte를 이어 쓰며,
		대상 파일이 없으면 새로 생성합니다.

		AppendAllText는 인코딩 오버로드를 제공하지 않고,
		대상 파일 기준으로 동작하도록 정리했습니다.

		AppendAllText는 BOM을 새로 기록하지 않습니다.

		대상 파일에 명확한 BOM이 있는 경우 해당 인코딩을 존중하고,
		BOM이 없거나 파일이 비어 있으면 CyphenEngine 기본 외부 텍스트 인코딩을 사용합니다.

		AppendAllText는 줄바꿈을 변환하지 않고
		LineEnding::Preserve 정책으로 입력 문자열의 줄바꿈을 유지합니다.

	2. FileSystem 단일 작업 API 확장
		FileSystem에 Create / GetSize / Copy / Move를 추가했습니다.

		Create는 파일 생성 자체만 담당하며,
		대상 파일이 이미 존재하면 실패하도록 정리했습니다.

		GetSize는 단일 파일 크기 조회만 담당하며,
		디렉터리에 대해서는 실패하도록 정리했습니다.

		Copy / Move는 sourcePath, targetPath 기준으로 동작합니다.

		기본 오버로드는 기존 대상을 교체하지 않고,
		canReplace 오버로드를 통해 대상 교체 여부를 명시하도록 구성했습니다.

	3. 디렉터리 트리 작업 추가
		FileSystem에 MakeDirectoryTree / DeleteDirectoryRecursively를 추가했습니다.

		MakeDirectoryTree는 Path 기준으로 경로를 분해하여
		중간 디렉터리를 순서대로 생성합니다.

		DeleteDirectory는 기존 정책대로 빈 디렉터리만 삭제합니다.

		하위 항목까지 삭제하는 작업은
		DeleteDirectoryRecursively로 명시적으로 분리했습니다.

	4. Separator 정책 계층 추가
		Separator / Separators를 추가하여 문자열 구분자 정책을 분리했습니다.

		Engine / Windows / Unix 기본 구분자를 제공하고,
		Refresh를 통해 사용자 구분자를 추가할 수 있도록 구성했습니다.

		Path는 고정된 '\' / '/' 비교 대신
		Separators에 등록된 구분자를 기준으로 경로 구분자를 판별합니다.

		PlatformFile Windows 구현은 Engine 구분자를 Windows 구분자로 변환한 뒤,
		WindowsString을 통해 Windows API용 문자열을 준비하도록 정리했습니다.

	5. PlatformFile 역할 확장
		PlatformFile에 ReadHead / AppendAllBytes / Create / GetSize /
		Copy / Move / DeleteDirectoryRecursively를 추가했습니다.

		ReadHead는 AppendAllText에서 BOM 확인에 필요한 파일 앞부분만 읽기 위해 추가했습니다.

		Windows 구현에서는 CreateFileW, GetFileAttributesExW,
		CopyFileW, MoveFileExW, FindFirstFileW / FindNextFileW를 사용해
		각 파일 시스템 작업을 처리합니다.

		PlatformFile은 File / FileSystem 뒤에 숨겨지는 private HAL로 유지했습니다.

		상위 엔진 계층과 사용자 코드는 PlatformFile을 직접 호출하지 않고,
		File / FileSystem 공개 API를 통해 접근합니다.

	6. CoreIOTests 확장
		CoreIOTests에 Separator 변환, AppendAllBytes / AppendAllText,
		FileSystem Create / GetSize / Copy / Move,
		MakeDirectoryTree / DeleteDirectoryRecursively 검증을 추가했습니다.

		AppendAllText가 없는 파일을 생성할 때 BOM을 기록하지 않는지 확인했습니다.

		Copy / Move의 기본 no-replace 정책과 canReplace 정책을 검증했습니다.

		DeleteDirectory와 DeleteDirectoryRecursively의 책임 분리를 테스트로 확인했습니다.

		최종 테스트 결과는 PASS=69 / FAIL=0입니다.


정리된 설계 기준
	- File은 파일 내용 I/O helper로 유지합니다.
	- FileSystem은 주어진 경로에 대한 파일 시스템 관리 작업을 담당합니다.
	- File과 FileSystem은 서로를 소비하지 않는 형제 계층입니다.
	- File과 FileSystem은 PlatformFile을 통해 실제 플랫폼 파일 접근을 수행합니다.
	- PlatformFile은 File / FileSystem 뒤에 숨겨지는 private HAL입니다.
	- Append는 일반 저장 모델이 아니라 명시적 누적 기록 primitive입니다.
	- 기본 저장 모델은 whole-file overwrite입니다.
	- File byte buffer는 CChar 정책과 독립적인 uint8 기반으로 유지합니다.
	- Separator는 Path 전용이 아니라 Core 문자열 구분자 정책 계층입니다.
	- DeleteDirectory는 빈 디렉터리 삭제 API로 유지합니다.
	- 하위 항목 삭제는 DeleteDirectoryRecursively로 명시적으로 분리합니다.
