Branch: #2-Renderer-모듈-개발

작업 요약
	Renderer DLL 모듈을 연결하기 위한 실행 컨텍스트와
	동적 모듈 관리 기반을 추가했습니다.

	LaunchContext / EngineContext를 통해 플랫폼 실행 정보를
	엔진 초기화 경계로 전달하고 보관하도록 구성했습니다.

	ModuleManager와 플랫폼별 ModuleLoader를 추가하고,
	CyphenRendererDx11을 독립 DLL 프로젝트로 분리했습니다.


진행한 커밋
	#2_1 [Refactor] 엔진 컨텍스트 추가 및 실행 컨텍스트 주입
	#2_2 [Module] 동적 모듈 관리 기반 및 Renderer ABI 계약 추가


주요 정리 내용
	1. LaunchContext / EngineContext 추가
		LaunchContext는 플랫폼 Launch에서 수집한
		주 윈도우 핸들과 초기 클라이언트 크기를 전달합니다.

		EngineContext는 전달받은 실행 정보를
		CyphenEngine 수명 동안 보관합니다.

		EngineStatus는 기존대로 CyphenEngine이 직접 소유합니다.

	2. ModuleDescriptor 추가
		설치된 모듈의 이름과 활성 상태를 보관합니다.

		목록에 포함된 항목은 설치된 모듈로 취급하고,
		isEnabled는 Preference에 따른 ON / OFF 상태로 사용합니다.

	3. ModuleManager 추가
		Refresh를 통해 활성 모듈을 로드하고,
		비활성 또는 제거된 모듈을 언로드합니다.

		로드된 모듈의 핸들과 로드 순서를 내부에서 관리하며,
		Shutdown에서는 역순으로 언로드합니다.

		GetLoadedModuleNames는 내부 목록의 참조가 아니라
		복사본을 호출자에게 전달합니다.

	4. Windows ModuleLoader 추가
		플랫폼별 동적 라이브러리 입출력을
		ModuleLoader 내부 구현으로 분리했습니다.

		Windows 구현은 LoadLibraryW / GetProcAddress /
		FreeLibrary를 사용합니다.

		ModuleLoader는 모듈 종류나 API 버전을 판단하지 않고,
		물리적인 로드, 심볼 조회, 언로드만 담당합니다.

	5. Renderer DLL ABI 계약 추가
		RendererModuleApi와 GetRendererModuleApi 진입점을 추가했습니다.

		DLL 경계에서는 C++ 추상 클래스나 vtable을 노출하지 않고,
		extern C 진입점과 고정된 API 구조체를 사용합니다.

		Renderer는 로드된 모듈의 API 버전과
		RendererType을 검증하도록 구성했습니다.

	6. CyphenRendererDx11 프로젝트 추가
		CyphenRendererDx11을 독립 DynamicLibrary 프로젝트로 추가했습니다.

		GetRendererModuleApi를 export하여
		Renderer API 버전과 Dx11 타입 정보를 제공합니다.

		구체 Dx11 구현 파일은 엔진 프로젝트에 중복 배치하지 않습니다.

	7. Launch 모듈 수명 연결
		LaunchContext에 ModuleDescriptor 목록을 추가했습니다.

		CyphenEngine 초기화 후 ModuleManager::Refresh를 호출하고,
		엔진 스레드 종료 후 ModuleManager::Shutdown을 호출합니다.

		모듈 로드 실패는 엔진 초기화 실패로 취급하지 않습니다.

	8. 빌드 경로 정리
		엔진과 Renderer DLL의 산출물을
		BuildArtifacts 공통 경로로 정리했습니다.

		실행 위치가 변경되어도 CoreIo 테스트 리소스를 읽을 수 있도록
		Debug 빌드 후 리소스 복사 단계를 추가했습니다.

	9. Renderer 검증 리소스 추가
		Resources/Thumnail에 프사.jpg와 프사2.jpg를 추가했습니다.

		이후 텍스처 출력과 회전 동작 검증에 사용합니다.


정리된 설계 기준
	- LaunchContext는 엔진 초기화 시점의 외부 입력을 전달합니다.
	- EngineContext는 엔진 실행 환경만 보관합니다.
	- ModuleDescriptor는 설치 모듈의 선언 정보와 활성 상태를 보관합니다.
	- ModuleManager는 모듈 상태와 수명을 관리합니다.
	- ModuleLoader는 플랫폼별 동적 라이브러리 입출력만 담당합니다.
	- Renderer는 Renderer ABI 조회와 검증을 담당합니다.
	- Renderer는 DLL을 직접 로드하거나 언로드하지 않습니다.
	- DLL 경계에는 C++ 추상 클래스와 vtable을 노출하지 않습니다.
	- vcxproj / filters는 Visual Studio 관리 파일이므로 탭 / LF 규칙에서 제외합니다.


검증
	CyphenRendererDx11 DLL 단독 빌드 성공을 확인했습니다.

	BuildArtifacts 출력 경로에서
	CyphenRendererDx11.dll이 로드되는 것을 확인했습니다.

	Renderer 실행 초기화와 프레임 출력은 아직 연결하지 않았습니다.

	- Linux .so 모듈 포팅
