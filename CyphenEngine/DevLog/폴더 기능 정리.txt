Build
	빌드 시스템 규약 및 설정을 관리합니다.

	- 플랫폼 타겟 정의
	- Debug / Release 설정
	- 빌드 타겟 구분
	- 모듈 활성화 여부
	- Export / Import 규약
	- CChar / CString 빌드 정책
	- 플랫폼 시스템 헤더 설정

	예:
	- TARGET_PLATFORM_WINDOWS
	- PLATFORM_WINDOWS
	- BUILD_TARGET_EDITOR
	- CCHAR_IS_UTF8
	- CCHAR_IS_UTF16
	- CCHAR_IS_WCHAR
	- CTEXT
	- Windows.h


Core
	Runtime과 Editor 양쪽에서 사용하는
	OS 비종속 엔진 기반 계층입니다.

	- Types / Enum 정의
	- CChar / CString 타입 확정
	- CPrimitiveTypes
	- Logger
	- Path
	- File 공개 API
	- TextCodec
	- Time 정책 계층
	- 공통 Utility
	- 공통 선언 및 구현

	주의:
	Core는 Windows/Linux API를 직접 호출하지 않습니다.
	Core는 Build 계층에서 확정된 플랫폼 / 문자열 규약을 사용만 합니다.

	Core의 Path는 순수 경로 문자열 유틸리티입니다.
	프로젝트 루트, Content, Saved, Logs 같은 엔진 정책 경로는 Path가 관리하지 않습니다.

	Core의 File은 단일 파일 동기 I/O 공개 API입니다.
	실제 플랫폼 파일 접근은 PlatformFile 내부 HAL 구현이 담당합니다.


HAL
	Core 내부 구현과 Platform 구현 사이의 내부 경계입니다.

	- PlatformFile
	- PlatformTime
	- Launch

	주의:
	HAL/Public은 외부에 공개해야 하는 최소 계약만 둡니다.
	PlatformFile처럼 File.cpp 뒤에 숨겨지는 내부 구현 경계는 HAL/Private에 둡니다.


Editor
	CyphenEngine Editor Runtime 계층입니다.

	- PIE
	- SceneView
	- GameView
	- Inspector
	- Content Browser

	Editor는 Runtime을 사용하지만,
	Runtime은 Editor를 알지 못합니다.


Engine
	CyphenEngine 최상위 엔진 인스턴스 계층입니다.

	- CyphenEngine
	- 엔진 상태 관리
	- 초기화 / 실행 / 종료 흐름

	Launch 계층만 CyphenEngine의 생성과 종료를 제어합니다.


Modules
	선택적으로 포함 가능한 엔진 기능 모듈입니다.

	- Renderer
	- Audio
	- Physics
	- Input

	빌드 설정에 따라 활성화 여부가 결정됩니다.


Platform
	플랫폼별 진입점 및 OS 종속 구현 계층입니다.

	Windows:
	- wWinMain
	- QueryPerformanceCounter
	- CreateFileW
	- ReadFile
	- WriteFile
	- MultiByteToWideChar

	Linux:
	- main
	- clock_gettime
	- open
	- read
	- write

	플랫폼별 cpp 구현은 빌드 타겟에 따라 선택됩니다.

	Platform 계층은 Core가 선언한 정책을 실제 OS API 호출로 연결합니다.
	Windows API 문자열 경계는 WindowsString이 담당합니다.

Test
	Debug 빌드에서 엔진 내부 기준선을 검증하는 테스트 계층입니다.

	- CoreIo 테스트
	- 회귀 확인용 최소 테스트 코드
	- Launch 단계에서 엔진 스레드 시작 전 실행
	- Runtime / Engine 및 Release 빌드에서는 의존하지 않습니다.
	- 유일한 호출자는 Launch의 _DEBUG 게이트입니다.
	- 테스트 실패는 실행 흐름을 차단하지 않고 OutputDebugString으로만 보고합니다.

	주의:
	테스트 등록기 / 리플렉션 / xUnit 류 프레임워크화를 하지 않습니다.
	FileSystem / ResourceManager / PlatformKnownPath 같은 미확정 계층을 테스트 편의를 위해 앞당기지 않습니다.


Resource
	애플리케이션 리소스 파일을 관리합니다.

	- .rc
	- .ico
	- Resource.h


Runtime
	게임 실행을 담당하는 엔진 본체입니다.

	- GameRuntime
	- World
	- Scene
	- Tick / Render 루프

	Runtime은 게임 실행 로직만 담당하며,
	Editor 계층에는 의존하지 않습니다.


pch
	사전 컴파일 헤더 및 공통 빌드 환경입니다.

	- Build 기본 include
	- 디버그 메모리 추적 설정

	주의:
	표준 라이브러리는 각 기능 구현 파일이 직접 include하는 것을 원칙으로 합니다.
