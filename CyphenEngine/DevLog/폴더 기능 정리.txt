CyphenEngine
	엔진 실행 파일 프로젝트입니다.
	엔진 본체와 플랫폼 진입점, 공통 기반 계층을 포함합니다.

	/DevLog
		개발 기록, 구조 메모, 작업 TODO를 관리합니다.

	/Resources
		실행 시 사용하는 테스트 / 샘플 파일 리소스를 관리합니다.

	/Source
		엔진 실행 파일의 실제 소스 루트입니다.

		/Build
			빌드 시스템 규약 및 설정을 관리합니다.

			- 플랫폼 타겟 정의
			- Debug / Release 설정
			- 빌드 타겟 구분
			- 모듈 활성화 여부
			- Export / Import 규약
			- CChar / CString 빌드 정책
			- 플랫폼 시스템 헤더 설정

		/Content
			엔진이 사용하는 콘텐츠 파일 디코딩 경로를 관리합니다.

			- 공통 Codec 계약
			- Image decode
			- JPEG decode
			- Windows / Linux 플랫폼별 JPEG leaf

			주의:
			Content는 파일 포맷 decode까지만 담당합니다.
			GPU resource 생성과 업로드는 Renderer 모듈 책임입니다.

		/Core
			Runtime과 Editor 양쪽에서 사용하는
			OS 비종속 엔진 기반 계층입니다.

			- Types / Enum 정의
			- CChar / CString 타입 확정
			- CPrimitiveTypes
			- Logger
			- Path
			- File / FileStream / FileSystem
			- TextCodec
			- Thread
			- Time 정책 계층
			- 공통 Utility

			주의:
			Core는 Windows/Linux API를 직접 호출하지 않습니다.
			Core는 Build 계층에서 확정된 플랫폼 / 문자열 규약을 사용만 합니다.

		/Engine
			CyphenEngine 최상위 엔진 인스턴스 계층입니다.

			- CyphenEngine
			- EngineContext
			- LaunchContext 수신
			- 엔진 상태 관리
			- 초기화 / 실행 / 종료 흐름
			- 활성화된 모듈 초기화 요청

			Launch 계층만 CyphenEngine의 생성과 종료를 제어합니다.
			Engine은 화면 출력 자체를 소유하지 않습니다.

		/HAL
			Core 내부 구현과 Platform 구현 사이의 내부 경계입니다.

			- PlatformFile
			- PlatformPath
			- PlatformTime
			- ModuleLoader
			- Launch
			- NativeWindowInfo

			주의:
			HAL/Public은 외부에 공개해야 하는 최소 계약만 둡니다.
			PlatformFile처럼 File.cpp 뒤에 숨겨지는 내부 구현 경계는 HAL/Private에 둡니다.

		/Modules
			엔진 내부 모듈 계약과 런타임 바인딩을 관리합니다.

			- ModuleDescriptor
			- ModuleBinder
			- ModuleManager
			- Command / CommandBuffer
			- Renderer 엔진 측 모듈
			- ResourceCommand 계약

			주의:
			Modules 계층은 엔진 내부에서 모듈을 선택하고 바인딩하는 쪽입니다.
			DLL / SO 내부의 구체 구현은 솔루션 루트의 Modules 하위 프로젝트에 둡니다.

		/Platform
			플랫폼별 진입점 및 OS 종속 구현 계층입니다.

			Windows:
			- wWinMain
			- QueryPerformanceCounter
			- CreateFileW
			- ReadFile
			- WriteFile
			- MultiByteToWideChar
			- LoadLibrary / GetProcAddress

			Linux:
			- main
			- clock_gettime
			- open
			- read
			- write
			- dlopen / dlsym

			플랫폼별 cpp 구현은 빌드 타겟에 따라 선택됩니다.
			Platform 계층은 Core가 선언한 정책을 실제 OS API 호출로 연결합니다.

		/Resource
			엔진 런타임 리소스 타입과 CPU-side 리소스 표현을 관리합니다.

			- ResourceId
			- Texture2D
			- TextureFormat

			주의:
			Resource 계층은 GPU API를 알지 않습니다.
			GPU resource 생성과 업로드는 Renderer 모듈 책임입니다.

		/Test
			Debug 빌드에서 엔진 내부 기준선을 검증하는 테스트 계층입니다.

			- CoreIo 테스트
			- Module 테스트
			- 회귀 확인용 최소 테스트 코드

			주의:
			유일한 호출자는 Launch의 _DEBUG 게이트입니다.
			Runtime / Engine 및 Release 빌드에서는 의존하지 않습니다.

	pch
		사전 컴파일 헤더 및 공통 빌드 환경입니다.

Docs
